# Workflow 10 — Entrega de Feature

## Objetivo

Entregar uma funcionalidade de forma incremental, testável e documentada.

## Passos

1. Definir critérios de aceitação.
2. Mapear frontend, backend, dados, segurança e testes afetados.
3. Implementar a menor alteração funcional completa.
4. Atualizar testes ou criar validação proporcional.
5. Atualizar documentação quando comportamento, configuração ou uso mudarem.
6. Atualizar `tasks/todo.md` e `CHANGELOG.md`.
7. Rever diff final.

## Proibido

- Misturar refactor estrutural não pedido.
- Introduzir dependências sem justificação.
- Declarar feature pronta sem validação.
