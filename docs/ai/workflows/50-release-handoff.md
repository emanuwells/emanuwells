# Workflow 50 — Release e Handoff

## Objetivo

Preparar entrega verificável e continuável noutra sessão ou ferramenta.

## Passos

1. Confirmar versão em `VERSION`.
2. Confirmar `CHANGELOG.md` append-only.
3. Confirmar README e comandos.
4. Confirmar testes/build/lint ou limitações.
5. Confirmar riscos e rollback.
6. Atualizar `docs/ai/ops/HANDOFF.md` se a tarefa tiver continuidade.

## Saída

- versão;
- resumo;
- validações;
- riscos;
- rollback;
- próximos passos.
