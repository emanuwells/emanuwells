# Decisões Técnicas

Registo de decisões permanentes ou com impacto arquitetural.

Este ficheiro não substitui o `CHANGELOG.md`, `tasks/todo.md` nem `tasks/lessons.md`.

## Quando Registar

Registar quando a decisão envolver:

- arquitetura;
- framework;
- base de dados;
- autenticação/autorização;
- integração externa;
- infraestrutura;
- padrão transversal;
- trade-off técnico relevante;
- rejeição explícita de alternativa técnica.

## Formato

```md
## DEC-YYYYMMDD-001 — <título>

**Data:** YYYY-MM-DD  
**Estado:** Proposta | Aceite | Substituída | Rejeitada  
**Responsável:** IA | Utilizador | Equipa

### Contexto

<problema ou necessidade>

### Decisão

<decisão tomada>

### Alternativas Consideradas

- <alternativa>: <motivo para aceitar/rejeitar>

### Consequências

- <impacto positivo>
- <trade-off ou risco>

### Refs

- <ficheiros, issues, documentação ou N/A>
```
